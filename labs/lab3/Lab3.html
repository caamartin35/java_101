  5. Bonus/Optional: Simple Calculator Extensions
    Here are two extensions you may add to the simple calculator for bonus.  If you plan to do these, first commit your regular calculator solution to git. This way, if you screw this up beyond belief, you wont lose all the work you did before.
    1. Backspace
      Add the ability for the user to enter a backspace, which undoes the last letter typed.  The user should be able to backspace multiple times, back through the equals sign (if entered), back through the right-hand side, back through the operator, and finally back through the left-hand side.  If the user enters backspace with no left-hand side, an error message should be displayed.
      Hint:  JComponentWithEvents defines the BACK_SPACE constant, which you can use to test the key against in the keyPressed() method.
       
    2. Two Operators with Precedence
      Add the ability to handle expressions with two operators, such as "1+2*3=".  You should respect order of operations, so "1+2*3=" evaluates to 7, but "1*2+3=" evaluates to 5.  You should also still support just one operator, as in "1+2=".

    b.  Ball Around Square
    In the file BallAroundSquare.java, write the animation in the ballAroundSquare.jar file. Right click that file in IntelliJ and select 'Run ballAroundSquare.jar'
      
    Notes:

    • The blue box is the largest square you can draw with a 10-pixel margin around it (so you will take the minimum of the width and height when computing the size of the square)
    • The circle pauses on the corners and is red while it pauses.
    • Hint:  the animation itself does not pause when the circle is on the corner -- the timer keeps firing away as normal.  Only your logic must basically ignore the timer for a little while at the right times.  To do this, perhaps you might introduce a counter (maybe called the "pauseCounter") that is set to 0 when you are not paused.  When you want to pause, set the counter to some positive value (say, 10, to start).  Now, when timerFired is called, if your pause counter is positive, just decrement it and return immediately.  Thus, you will ignore some number (again, say 10, to start) of calls to timerFired, and only when the pauseCounter reaches 0 will you again start processing the timer events.
    • Also, you will want to use the page.drawString method to draw the text.  It takes 3 parameters -- a String, and an x and y location at which to draw the string.  The x is the left edge of the string and the y is the "baseline", which is near the bottom of the string.  For example, try this line of code:
         page.drawString("Carpe diem", 50, 100);
      Then, read the online API for the drawString method in the Graphics class for more details.
    • You will also want to use the substring method of the String class.  This method takes 2 parameters -- the starting index (inclusive) and the ending index (exclusive), and returns the substring between them.  For example, "abcdefg".substring(2,5) returns the string "cde".
    • Another hint:  you may want to draw two strings, one per visible line, using different substrings of a third string.
